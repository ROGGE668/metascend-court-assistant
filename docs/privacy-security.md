# 隐私与安全设计

## 设计原则

1. **本地优先**: 默认所有语音、文本、模型推理都在用户设备上完成。
2. **最小权限**: 应用只申请麦克风和蓝牙权限。
3. **加密存储**: 录音和日志默认 AES-256 加密。
4. **透明可控**: 用户可随时暂停、删除数据。

## 数据处理

### 音频数据

- 麦克风采集的音频流实时处理，不主动上传云端。
- 可选录音模式：开启后音频写入 `data/recordings/`，默认加密。
- 关闭录音模式时，音频缓冲在内存中，会话结束后释放。

### 文本数据

- 转写文本、策略建议、日志均本地存储。
- 调试日志不包含完整语音波形或敏感案情细节。
- 知识库检索不依赖外部服务。

### 模型与下载

- 模型首次从 Hugging Face / 镜像下载后缓存本地，后续离线运行。
- 下载脚本 `scripts/download_models.py` 支持通过 `HF_ENDPOINT` 配置镜像。
- 模型缓存目录：`~/.cache/metascend/models`。

## 加密方案

### 录音文件加密

```python
from cryptography.fernet import Fernet

key = user_password_derived_key  # 由用户密码通过 PBKDF2 派生
fernet = Fernet(key)
encrypted = fernet.encrypt(audio_bytes)
```

- 录音文件扩展名 `.enc`。
- 密钥不存储在磁盘上，由用户每次输入密码派生。
- 支持生物识别解锁时，使用 Secure Enclave 存储密钥（macOS 平台）。

### 日志加密

- 敏感日志写入 SQLite 时启用 SQLCipher 加密。
- 普通运行日志可保留为明文，但不得包含案情细节。

## 权限模型

| 权限 | 用途 | 是否必需 |
|------|------|----------|
| 麦克风 | 采集法庭语音 | 是 |
| 蓝牙 | 连接耳机 | 否（可用有线耳机或有线音箱） |
| 通知 | 状态提示 | 否 |
| 定位/通讯录/照片 | 不使用 | 否 |

## 合规对照

| 要求 | 实现方式 |
|------|----------|
| 《个人信息保护法》最小必要 | 仅采集必要音频，不收集无关信息 |
| 用户知情同意 | 首次启动明确告知数据本地处理 |
| 用户删除权 | 提供“清除所有本地数据”按钮 |
| 数据安全 | AES-256 加密 + 本地存储 |
| 法庭电子设备规定 | 提供完全静默模式、一键静音 |

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| 设备丢失导致录音泄露 | 录音加密，无密码无法解密 |
| 应用被恶意提取模型 | 模型缓存使用标准 Hugging Face 格式，不存储额外敏感信息 |
| 调试日志泄露案情 | 生产环境关闭详细日志；日志不包含完整转写 |
| 用户误信 AI 建议 | UI 强制显示免责声明；高风险建议过滤 |

## 审计与自检

- 提供 `--offline` 启动参数，禁止任何网络请求，用于法庭等敏感场景。
- 启动时检测是否有外部连接，若有则告警并退出。
- 定期生成隐私报告：数据量、加密状态、最近访问时间。
